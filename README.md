# gpt100m

This branch trains the current nano blended-data pipeline under the 100M parameter limit. The model uses RoPE attention, RMSNorm, SwiGLU MLPs, and tied token/output embeddings.

## Structure

```text
gpt100m/
  README.md
  model.py                         # Competition-compatible nano model and load_model()
  train.py                         # Config-driven training loop
  pipeline_colab.ipynb             # Colab training/eval/packaging workflow
  config/
    config.py                      # Nano blended-data training config
  data/
    fineweb.py                     # FineWeb-Edu tokenization script
    blend.py                       # FineWeb/Wikipedia/Papers/Books tokenization script
  checkpoints/                     # Generated by training; ignored by git
  results/                         # Generated by notebook runs; ignored by git
```

## Setup

Install the runtime dependencies in your Python environment:

```bash
pip install torch numpy tiktoken datasets tqdm
```

## Collect Data

`data/blend.py` downloads the training blend, tokenizes it with the GPT-2 tokenizer, and writes raw uint16 train shards under `data/blend/<source>`:

```bash
python data/blend.py --data-root data/blend --streaming
```

The default blend sources are:

```text
FineWeb-Edu:  HuggingFaceFW/fineweb-edu, sample-10BT, text
Wikipedia:    wikimedia/wikipedia, 20231101.en, text
Papers:       armanc/scientific_papers, arxiv/pubmed, article
Books:        incredible45/Gutenberg-BookCorpus-Cleaned-Data-English
```

Training samples each optimizer step as 16 FineWeb-Edu, 7 Wikipedia, 5 Papers, and 5 Books microbatches. The Papers source uses arXiv/PubMed at 70/30 over time. Validation during training reads the TA-style root `val.bin`, not a FineWeb-Edu validation shard.

Generated train shards look like:

```text
data/blend/fineweb_edu/fineweb_train_000000.bin
data/blend/wikipedia/wikipedia_train_000000.bin
data/blend/papers_arxiv/papers_arxiv_train_000000.bin
data/blend/papers_pubmed/papers_pubmed_train_000000.bin
data/blend/books/books_train_000000.bin
```

Generated shards should stay untracked.

Each source writes a cursor file next to its shards, for example `fineweb_prepare_state.json`. To append more shards without deleting the old ones, resume from that state:

```bash
python data/blend.py --data-root data/blend_new --streaming --sources fineweb --shard-size 100000000 --max-shards-per-source 5 --start-shard-index <next_shard_index> --skip-hf-items <next_hf_item_index> --skip-tokens-in-first-item <next_item_token_offset>
```

For a 20,000-step run at the default batch size, the loader consumes about 8.11B tokens total. With the no-PubMed PG19 blend and 100M-token shards, targets around FineWeb-Edu 40, Wikipedia 18, arXiv 13, and PG19 13 avoid cycling each source more than once.

For a clean no-PubMed rebuild that leaves the old dataset untouched, write to `data/blend_new` and point the training config there:

```bash
python data/blend.py --data-root data/blend_new --streaming --sources fineweb --shard-size 100000000 --max-shards-per-source 40 --overwrite
python data/blend.py --data-root data/blend_new --streaming --sources wikipedia --shard-size 100000000 --max-shards-per-source 18 --overwrite
python data/blend.py --data-root data/blend_new --streaming --sources arxiv --shard-size 100000000 --max-shards-per-source 13 --overwrite
python data/blend.py --data-root data/blend_new --streaming --sources pg19 --shard-size 100000000 --max-shards-per-source 13 --overwrite
```

Those are the per-source targets for the 20,000-step no-PubMed blend: FineWeb-Edu 40, Wikipedia 18, arXiv 13, and PG19 13.

## Train

Single GPU or CPU:

```bash
python train.py --config config/config.py
```

DDP multi-GPU:

```bash
torchrun --standalone --nproc_per_node=8 train.py --config config/config.py
```

Training writes `checkpoints/checkpoint.pt` as a raw PyTorch state dict, plus `checkpoint_metadata.pt` and `train.log`.

## Checks

Parameter check:

```bash
python model.py
```

The current config has `99,764,784` parameters.
