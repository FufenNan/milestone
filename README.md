# gpt100m

This branch trains the current nano blended-data pipeline under the 100M parameter limit. The model uses RoPE attention, RMSNorm, SwiGLU MLPs, and tied token/output embeddings.

## Structure

```text
gpt100m/
  README.md
  model.py                         # Competition-compatible nano model and load_model()
  train.py                         # Config-driven training loop
  pipeline_colab.ipynb             # Colab training/eval/packaging workflow
  config/
    config.py                      # Nano blended-data training config
  data/
    fineweb.py                     # FineWeb-Edu tokenization script
    blend.py                       # FineWeb/Wikipedia/arXiv/PG19 tokenization script
  checkpoints/                     # Generated by training; ignored by git
  results/                         # Generated by notebook runs; ignored by git
```

## Setup

Install the runtime dependencies in your Python environment:

```bash
pip install torch numpy tiktoken datasets tqdm
```

## Collect Data

`data/blend.py` downloads the training blend, tokenizes it with the GPT-2 tokenizer, and writes raw uint16 train shards under `data/blend/<source>`:

```bash
python data/blend.py --data-root data/blend --streaming
```

The default blend sources are:

```text
FineWeb-Edu:  HuggingFaceFW/fineweb-edu, sample-10BT, text
Wikipedia:    wikimedia/wikipedia, 20231101.en, text
Papers:       armanc/scientific_papers, arxiv, article
PG19:         emozilla/pg19, text
```

Training samples each optimizer step as 16 FineWeb-Edu, 7 Wikipedia, 5 arXiv Papers, and 5 PG19 microbatches. Validation during training reads the TA-style root `val.bin`, not a FineWeb-Edu validation shard.

Generated train shards look like:

```text
data/blend/fineweb_edu/fineweb_train_000000.bin
data/blend/wikipedia/wikipedia_train_000000.bin
data/blend/papers_arxiv/papers_arxiv_train_000000.bin
data/blend/pg19/pg19_train_000000.bin
```

Generated shards should stay untracked.

To append more shards without deleting the old ones, use `--append-existing` and a total target:

```bash
python data/blend.py --data-root data/blend --streaming --sources fineweb --target-shards-per-source 40 --append-existing --shuffle-seed 20260528
```

For a 20,000-step run at the default batch size, the loader consumes about 8.11B tokens total. With the no-PubMed PG19 blend and 100M-token shards, targets around FineWeb-Edu 40, Wikipedia 18, arXiv 13, and PG19 13 avoid cycling each source more than once. If a previous exact dataset cursor was not saved, `--shuffle-seed` samples a different stream order for the appended shards.

## Train

Single GPU or CPU:

```bash
python train.py --config config/config.py
```

The default config uses a 12,000-step LR schedule for the first session. Continue training with `config/config_continue.py`, which starts a separate low-LR phase from the 12,000-step checkpoint. In Colab, the notebook passes `--steps 12000` for the first run and mirrors each saved checkpoint to Drive under the run's `live_checkpoints/` folder.

DDP multi-GPU:

```bash
torchrun --standalone --nproc_per_node=8 train.py --config config/config.py
```

Training writes `checkpoints/checkpoint.pt` as the best full training checkpoint, plus `checkpoint_metadata.pt` and `train.log`.
