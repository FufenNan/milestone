# gpt100m

The baseline keeps the `build-nanogpt/train_gpt2.py` GPT-2 architecture but scales it to `n_layer=8, n_head=12, n_embd=768`, which is 96M.

## Structure

```text
gpt100m/
  README.md
  model.py              # Competition-compatible model and load_model()
  train.py              # Config-driven training loop
  baseline/
    model.py            # Copy of the baseline model definition
  config/
    baseline.py         # Baseline model, data, optimizer, and checkpoint config
  data/
    fineweb.py          # Dataset collection/tokenization script only
  checkpoints/          # Generated by training; do not submit large experiments to git
```

## Setup

Install the runtime dependencies in your environment:

```bash
pip install torch numpy tiktoken datasets tqdm
```

## Collect Data

`data/fineweb.py` downloads FineWeb-Edu `sample-10BT`, tokenizes with the GPT-2 BPE tokenizer, and writes `.npy` shards. The first shard is named as validation data and the rest are training data.

```bash
cd /home/kaiying/llm/gpt100m
python data/fineweb.py --output-dir data/edu_fineweb10B
```

The `data/` folder should contain scripts only in the repo. Generated shards should stay untracked.

## Modify Config

Edit `config/baseline.py` for experiments:

- `data_dir`: directory containing tokenized `.npy` shards.
- `checkpoint_dir`: output directory for checkpoints.
- `block_size`, `vocab_size`, `n_layer`, `n_head`, `n_embd`: model shape. Keep `vocab_size=50257` and `block_size=1024` for the competition evaluator.
- `total_batch_size`, `micro_batch_size`: effective token batch size and per-process micro batch size.
- `max_steps`, `warmup_steps`, `max_lr`, `min_lr`, `weight_decay`: optimizer and schedule.
- `eval_interval`, `eval_iters`, `checkpoint_interval`: validation and checkpoint cadence.

After changing model shape, run the parameter check:

```bash
python model.py
```

The model must stay at or below `100,000,000` parameters.

## Train

Single GPU or CPU:

```bash
python train.py --config config/baseline.py
```

DDP multi-GPU:

```bash
torchrun --standalone --nproc_per_node=8 train.py --config config/baseline.py
```

Training writes `checkpoints/checkpoint.pt` plus optional step checkpoints. `checkpoint.pt` is saved as a raw PyTorch state dict so it can be used directly by the competition evaluator.

## Submission Files

For a HuggingFace submission, upload at least:

```text
checkpoint.pt
model.py
config/baseline.py
```

`model.py` defines the required function:

```python
def load_model(checkpoint_path: str, device: str = "cuda") -> torch.nn.Module:
```

The returned model accepts GPT-2 token IDs of shape `(batch, sequence_length)` and returns logits of shape `(batch, sequence_length, 50257)`.
